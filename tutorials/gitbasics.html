Git Basics

First configuration

git config --global user.name "John Smith"
git config --global user.email "johnsmith@johnson.com"
SSH Keys

Type in the following command to see if any existing SSH keys are present:

ls -al ~/.ssh

If not:

ssh-keygen -t ed25519 -C "johnsmith@johnson.com"

Check if SSH Agent is running

eval "$(ssh-agent -s)"

Add your SSH private key to the ssh-agent. If you created your key with a different name, or if you are adding an existing key that has a different name, replace id_ed25519 in the command with the name of your private key file.

ssh-add ~/.ssh/id_ed25519

Copy the contens of the id_ed25519.pub file to your clipboard

clip < ~/.ssh/id_ed25519.pub

Go to github profile picture / Settings / Access / SSH and GPG keys / New SSH Key

Add a descriptive title like "Lenovo Notebook"

Paste your key into the “Key” field.

You are done!

Upload an existing local file

Open git bash in your local folder

git init

We create an empty repository in github.com

git commit -m "first commit"
git remote add origin https://github.com/link-to-repository
git push -u origin master
Clone existing file in github

Fork the repository in github

git clone https://github-code-ssh-link
Change repository branch
  1. git branch (see branches)
  2. git checkout -b stage (moves current git to a newly created branch called stage)
  3. git add .
  4. git commit -m "message"
  5. git push origin stage
Upgrade changes

  1. git status
  2. git add . (or name of the specific files)
  3. git commit -m "Different message"
  4. git push origin master (or the name of the current branch)
